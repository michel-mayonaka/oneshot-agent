# Doc Audit Skill

目的:
- リポジトリ内のドキュメントを横断的に調査し、不整合を洗い出す。
- 調査結果を再現可能なレポートとしてまとめる。

必須チェック項目:
- 手順の齟齬（同目的で手順が異なる）
- コマンド不一致（存在しない/実態とズレたコマンド）
- パス/ファイル名不一致（参照先が存在しない/旧名）
- 設定/環境変数の矛盾（同名設定で説明/値が食い違う）
- 前提条件の矛盾（OS/バージョン/依存条件が一致しない）
- 重複/分岐の無秩序（同内容が複数箇所でズレる）

レポート要件:
- Markdown で出力する。
- 次のセクションを必ず含める。
  1) 概要
  2) 実行コマンド（実際に叩いたコマンド全文）
  3) 調査プロセス（手順の箇条書き）
  4) 不整合一覧（重大度/概要/根拠ファイル）
  5) 根拠詳細（該当ファイルの引用 or 要約）

記述ルール:
- 断定が難しい場合は「要確認」と明示する。
- 修正は行わず、調査結果のみを報告する。
- 重要度は High/Medium/Low を使う。
