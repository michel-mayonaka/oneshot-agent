# 目的
- ワンショット実行の入口を一本化します。
- 入出力とログの契約を固定します。
- 失敗時の人間復旧を前提にします。

# 非目的
- 常駐型のジョブ運用は対象外です。
- 秘密情報の安全保管は対象外です。
- 複数エージェント連携は対象外です。

# 想定ユーザー / ユースケース
- ドキュメント監査と修正の自動化。
- 小規模リファクタの一次実行。
- 反復作業のプロンプト運用。
- PR下書きのたたき台作成。

# 成功指標
- `make test-doc` が成功する。
- run ごとに `report.md` が生成される。
- `logs/events.jsonl` が残る。
- 失敗時に `stderr_and_time.txt` が残る。

# 用語集
- run_id: 1回の実行を識別するID。
- run_dir: `worklogs/<spec>/<run_id>` 配下。
- spec: `specs/*.yml` の実行定義。
- worktree: Git の作業コピー。
- prompt.raw.txt: ユーザ入力の原文。
- prompt.txt: skills 前置後の最終文。
- skills: プロンプト前置のガイド群。
- artifacts: 生成物の保存先。
- worklog: 実行ログの可読版。

# 保証
- 用語定義は本書を基準とします。
- 目的と非目的はレビュー基準です。

# 仮定
- 成功指標の計測は別途行います。
- 利用者が秘密情報を入力しません。
