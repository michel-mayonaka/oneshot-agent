# ユーザー向けドキュメント

## 目的
- Codex CLI エージェントを使用した GitHub Issue 駆動ワークフローの提供。
- 定型作業を YAML にてジョブ化する機能の提供。
  - Claude Code や Roo Code の /command に近い位置づけ。ただし違いもある。
  - 例: skill を明示的にバンドルできる。
  - 例: プロンプトを資産化できる（/command に近い）。
  - 例: PR / worktree / Issue の発行はハーネス側のスクリプトで提供。

## 利点
- 作業計画-実装-PR 発行-レビュー対応までを、人は主にレビューに専念できる。
  - Issue / branch / worktree / PR の発行はエージェントとハーネス側で行う。
  - 人は make コマンドを通じてエージェントとやり取りを行い、作業計画・実装のレビューを行う。
- エージェントの実行ログと成果物は実行単位で管理される。
  - エージェントの挙動に問題がある場合に、監査が容易。
- ワークフロー通りに作業を行うことで、目的が明確なセッションになり、ロングセッションになりづらい。
  - Issue から作業を行わせる場合は Issue の内容をもとにプロンプトをワンショットするのみで、セッションが終了する。
  - Issue 作成時は planning モードで対話的に計画を行うが、この際はロングセッションにならないよう Issue の粒度に注意が必要。
